Project info:

WiFi - No Webserver example


Project description:

The application contains a basic application that permits to use the Flyport module connectivity without the webserver.

In this example, the user firmware is very basic, it just connects to the default network using the command WFConnect(WF_DEFAULT) and awaits for the connection to be established (monitoring the system variable WFStatus). By default, Flyport creates an open ad hoc network, but all the parameters can be changed running the wizard.

Note:

With this configuration, the Flyport can be reached with "ping" but not with a browser.
